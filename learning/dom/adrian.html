  3. Video ketiga programmer zaman now:
    4. Document Object Model (DOM) adalah representasi objek dari struktur halaman web yang dibuat oleh browser saat sebuah website diload. DOM ini dibuat dalam bentuk pohon dan di tiap cabangnya terdapat objek-objek dari jenis elemen HTML yang ada di tampilan website. JavaScript memiliki kemampuan untuk memanipulasi DOM, sehingga aplikasi web yang dibuat bisa terlihat lebih dinamis.
    • DOM dibuat dalam bentuk pohon, dan di tiap cabangnya terdapat objek-objek dari jenis elemen HTML yang ada di tampilan website.
    • JavaScript memiliki kemampuan untuk memanipulasi DOM.
    • DOM bukanlah bagian dari bahasa pemrograman JavaScript, melainkan fitur dari W3C.
    • DOM dapat diintegrasikan dengan banyak bahasa pemrograman, bukan hanya JavaScript.
    • DOM digunakan untuk mengambil, memanipulasi, mengubah, menambahkan, atau menghapus elemen HTML pada sebuah website.
    • DOM membuat aplikasi web terlihat lebih dinamis dan interaktif.
    1. Tipe Data
      • DOM memiliki representasi data
      • Tidak terlalu banyak tipe data yang terdapat di DOM
      • Oleh karena itu, sebenarnya DOM sangat sederhana, namun penggunaanya sangat bermanfaat.
    2. Document
      3. Document merupakan representasi object dari halaman web, untuk mengakses document, kita bisa gunakan object document.
    3. Document Object
      • Document secara otomatis dibuat dalam browser ketika membuka halaman web, sehingga kita tidak perlu membuat objek Documen secara manual.
      • Kita bisa langsung menggunakan kata kunci dokument
    4. Document Property
      • Document memiliki banyak sekali property
      • Property dalam Document bisa kita gunakan untuk melihat semua data yang terdapat dalam document halaman web.
    5. Document Method
      • Document banyak sekali method
      • Banyak method yang terdapat di document digunakan untuk memanipulasi data DOM, misal membuat node. Element, atribut atau mengambil dan menyeleksi Node didalam document.
    6. Node
      • Node adalah base class dari Document, element dan attr
      • Artinya semua fitur yang dimiliki di node dimiliki turunannya
      • Node dalam DOM itu bentuknya Tree( pohon) artinya node kita bisa melihat parent (node diatasnya), children (node dibawahnya), sibling(node di sebelahnya).
    7. Node Property
      • Node memiliki banyak sekali Property
      • Property di Node banyak berisi informasi reference ke Node lain nya. Seperti ke node parentnya, node children, atau node sebelahnya.
    8. Node Method
      • Node memiliki banyak sekali method
      • Method di node bisa digunakan untuk memanipulasi data node di dalam Node tersebut, misal menambah/menghapus node children.
    9. Element
      10. Element merupakan node yang berbentuk element, biasanya adalah element HTML, misal ,,,
      ,, dan lain-lain.
    10. Element Property
      • Element memiliki banyak sekali property
      • Dan karena elemnt adalah turunan dari node, semua property Node pun bisa digunakan di Element
    11. Element Method
      • Element memiliki banyak sekali method
      • Dan karena element adalah turunan dari node, semua method Node pun bisa gunakan di Element
    12. Membuat Element
      • Sebelumnya kita hanya mengambil element menggunakan ide. Artinya kita perlu buat dulu element nya di HTML.
      • Kita juga bisa membuat object element baru, dan menambahkannya ke halaman web.
      • Kita bisa menggunakan method createElement(tag) pada Document
