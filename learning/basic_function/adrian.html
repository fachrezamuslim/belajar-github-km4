Video 1: Intro

function menurut Douglas Crockford adalah kunci utama pada javascript yang membuat javascript sangat powerfull

Function adalah sebuah sub-program/sub-routine yang dapat 'dipanggil' di bagian lain pada program dan merupakan struktur dasar pembentuk JavaScript. Function juga disebut sebagai prosedur (kumpulan statement untuk melakukan tugas atau menghitung suatu nilai). Untuk dapat menggunakannya, kita harus 'membuat' terlebih dahulu function tersebut, lalu 'memanggil'-nya. Function termasuk kedalam First-Class Object.Adapun alasan penggunaan function yaitu:

  1. Reusability (DRY: Don't Repeat Yourself)

    2. Reusability ialah penggunaan kembali dari code yang sudah dibuat

  2. Dekomposisi / Abstraksi
    3. Dekomposisi / Abstraksi digunakan untuk menyembunyikan kompeksitas dari program yang kita buat
  3. Modularitas
Function memiliki 2 kategori yaitu:
  1. Built-in Function: fungsi yang disediakan oleh JavaScript dan kita tinggal 'memanggil'-nya contohnya alert(), prompt(), confirm()
  2. User-defined Function: fungsi yang kita buat sendiri dengan menggunakan keyword function. Pada function terdapat nama function (optional), parameter/argument, dan body. User-defined function dapat mengembalikan nilai (return value) atau tidak. Adapun beberapa cara untuk membuat User-defined Function yaitu:
    • Dengan deklarasi / function declaration yang dapat dibuat dengan sintaks:
      function namaFunction(param1, param2) { body }
    • Dengan ekspresi / function expressions yang dapat dibuat dengan sintaks:
      var namaFunction = function (param1, param2) { body }

Adapun cara untuk memanggil function yaitu dengan menuliskan sintaks:
namaFunction()

Video 2: Membuat dan Menjalankan FUNCTION

ketika kita menggunakan sebuah function, kita akan memberikan sebuah nilai atau serangkaian nilai sebagai input. Function tersebut akan menggunakan nilai tersebut untuk melakukan suatu operasi atau tugas tertentu. Setelah melakukan operasi tersebut, function akan mengembalikan output yang dapat digunakan atau diproses lebih lanjut.

Sebuah function yang baik harus fokus pada satu tugas atau operasi tertentu. Hal ini membantu untuk membuat kode yang lebih mudah dipahami dan diorganisir. Ketika kode dikelompokkan dalam sebuah function, kita dapat dengan mudah memahami apa yang terjadi di dalamnya dan bagaimana ia bekerja.

Selain itu, function yang baik harus dapat dipanggil berulang kali (reusable). Artinya, kita dapat memanggil function tersebut kapan saja dan di mana saja dalam kode kita tanpa harus menulis ulang kode yang sama berulang kali. Hal ini membantu untuk menghemat waktu dan usaha.

Terakhir, ketika kita menggunakan function, kita dapat dengan mudah melacak bagaimana nilai input diolah dan bagaimana output dihasilkan. Hal ini membantu untuk mengurangi kesalahan dalam kode dan memudahkan perbaikan jika diperlukan di masa depan.

Jadi, intinya adalah sebuah function adalah sebuah blok kode yang melakukan tugas tertentu dan menerima input, memproses input tersebut, dan mengembalikan output. Sebuah function yang baik fokus pada satu tugas, dapat dipanggil berulang kali, dan mudah dilacak.

Video 3: PARAMETER dan ARGUMENT

parameter adalah variabel yang ditulis dalam kurung pada saat function dibuat dan digunakan untuk menampung nilai yang dikirimkan saat function dipanggil. Sementara itu, argument adalah nilai yang dikirimkan ke parameter saat fungsi dipanggil.

Penting untuk diingat bahwa variabel arguments juga tersedia dalam setiap function, yang akan menyimpan nilai array dari semua argument yang dikirimkan ke function. Hal ini memungkinkan kita untuk mengakses semua argument yang dikirim ke function, bahkan jika kita tidak menentukan parameter untuk setiap nilai.

Jika jumlah parameter saat function dibuat berbeda dengan jumlah argument yang dikirim saat memanggil function, maka beberapa hal dapat terjadi:

  1. Jika jumlah parameter lebih banyak dari jumlah argument, parameter yang tidak diisi akan bernilai undefined.
  2. Jika jumlah parameter lebih sedikit dari jumlah argument, argument tersebut tetap tersimpan dalam variabel arguments dan dapat diakses dari dalam function menggunakan teknik iterasi.

Contoh penulisan parameter saat membuat function adalah:

function namaFunction(parameter) { // perintah untuk melakukan tugas terhadap parameter }

Contoh penulisan argument saat memanggil function adalah:

namaFunction(argument)

Video 4: REFACTORING

Refactoring adalah sebuah proses mengubah kode agar lebih 'baik' tanpa mengubah fungsionalitasnya. Adapun beberapa faktor mengapa refactoring harus dilakukan yaitu:

  1. Readibility: kemudahan dalam membaca kode program

    2. Readability adalah kemampuan untuk dengan mudah membaca, memahami, dan mengelola kode program.

  2. DRY (Don't Repeat Yourself): duplikasi kode

    3. DRY adalah prinsip yang mengatakan bahwa kode tidak boleh diulang jika tidak perlu. Menghindari duplikasi kode dapat membantu untuk menjaga kekonsistenan kode, mengurangi kesalahan, dan memudahkan perawatan kode. Sebagai gantinya, kita dapat menggunakan fungsi atau metode yang dapat dipanggil kembali jika kita membutuhkan tindakan atau logika yang sama dalam beberapa bagian kode.

  3. Testability: penulisan koda agar mudah saat proses pengujian

    Testability adalah kemampuan kode untuk diuji dengan mudah. Semakin mudah kode diuji, semakin cepat kita dapat menemukan kesalahan dan memperbaikinya.

  4. Performance: bagaimana cara menulis kode yang dapat meningkatkan performa dari program yang dibuat

    5. Performance adalah kemampuan kode untuk berjalan dengan cepat dan efisien. Dalam penulisan kode, kita harus mempertimbangkan efisiensi dan skalabilitas

  5. Maintainability: bagaimada program dapat dengan mudah dikelola atau dikembangkan

Video 5: VARIABLE SCOPE

scope adalah aturan yang menentukan bagaimana sebuah variabel dapat diakses dalam program. Dalam bahasa pemrograman terdapat dua jenis scope yaitu:

  1. Block scope, digunakan pada bahasa pemrograman seperti C dan Java, variabel yang dibuat di dalam block scope hanya dapat digunakan di dalam block tersebut, dan tidak dapat diakses di luar dari block itu.
  2. Function scope, digunakan pada bahasa pemrograman seperti Javascript, variabel yang dibuat di dalam function scope dapat diakses di luar dari function tersebut.

Variabel yang dibuat tanpa dibungkus scope di dalam JavaScript disebut global scope atau window scope. Variabel global dapat diakses pada block scope mana pun. Namun, jika terdapat deklarasi variabel dengan nama yang sama dalam block scope atau saat menjadi parameter dengan nama yang sama dengan variabel global scope, maka akan terjadi penimpaan variabel dan menjadi dua variabel yang berbeda.

Video 6: DECLARATION vs. EXPRESSION

Function Declaration adalah cara tradisional dalam mendefinisikan fungsi di JavaScript, yang bisa diletakkan di mana saja dalam kode. Hal ini disebabkan karena konsep hoisting pada JavaScript, dimana deklarasi fungsi akan diangkat ke bagian atas dari lingkup saat dijalankan, sehingga fungsi dapat dipanggil sebelum deklarasi sebenarnya dalam kode.

Function Expression adalah cara lain untuk mendefinisikan fungsi di JavaScript, yang melibatkan definisi fungsi dalam sebuah ekspresi. Function Expression biasanya didefinisikan sebelum dipanggil, dan karena itu tidak dapat diakses sebelum definisi sebenarnya. Function Expression memiliki fleksibilitas lebih dalam hal penggunaan fungsi sebagai closure, fungsi sebagai argumen untuk fungsi lain, atau sebagai IIFE (Immediately Invoked Function Expression), di mana fungsi akan langsung dipanggil setelah didefinisikan.

Rumus dari Function Declaration yaitu:
function namaFunction(listParameter opt) { FunctionBody }

Rumus dari Function Expression yaitu:
function namaFunctionopt (listParameteropt) { FunctionBody }

Adapun perbedaan dari Function Declaration dan Function Expression yaitu:

  • Function Declaration:
    1. Lebih fleksibel (dapat ditulis dimanapun) karena pada javascript ada konsep Hoisting dimana deklarasi sebuah function pasti disimpan terlebih dahulu didalam memory
    2. Mudah dipahami pemula
  • Function Expression:
    1. Harus didefinisikan terlebih dahulu sebelum dipanggil
    2. Lebih powerfull:
      • Sebagai closure
      • Sebagai argumen untuk function lain
      • IIFE (Immediately Invoked Function Expression)
