Intro

Menurut Douglas Crockford “function adalah kunci utama pada javascript yang membuat javascript sangat powerfull”. Jika didalam bahasa pemrograman lain terdapat method, class, constructor dan modul pada javascript dapat menggunakan sebuah function. Function adalah sub program / sub-routine yang dapat di panggil di bagian lain dari program. Merupakan struktur dasar membentuk dari javascript. Dapat di sebut juga sebagai prosedur ( kumpulan dari statement yang melakukan tugas). Untuk dapat menggunakannya, kita harus membuat terlebih dahulu function tersebut, lalu memanggil nya. Termasuk kedalam First-Class Object.

Kelebihan function adalah karena function memiliki Reusability, untuk tidak menggunakan berulang DRY (Do not Repeat Yourself). Untuk Dekomposisi / abstraksi berfungsi untuk membagi bagi program. Ketika membagi bagian program akan memudahkan dalam mencari kesalahan (Modularitas).

Function di bagi menjadi 2 ada Built-in Function adalah function yang telah dibuat javascript. Built in function tinggal di panggil saja, contoh fungsi javascript aalah alert(), confirm() dan lain lain. Ada User Defined Functiom adalah sebuah function yang di definisikan oleh pengguna. Cara membuat function adalah dengan menggunkan keyword function. Lalu function dapat menggunakan nama dan tidak. Di function dapat menggunakan parameter / argument. Function body dibungkus dengan {}. Dan Function dapat mengembalikan value dan juga tidak. Function di bagi 2 ada function declaration dan function expretion.


Membuat dan Menjalankan

Cara kerja function adalah terdapat input yang masuk ke dalam function. Kemudian function akan mengolah nya. Lalu function akan menghasilkan output. Function yang baik ketika function hanya mengerjakan satu hal saja yang artinya blok kode ini dibuat untuk melakukan tugas spesifik. Berfungsi untuk dapat di panggil berulang kali, memudahkan penelusuran jika terjadi kesalahan.

Cara membuat function adalah menambahkan keyword function yang dapat menggunakan nama di batasi dengan {}. DI sebelah nama terdapat () bertujuan menyimpan argumen/ parameter. Lalu di dalam function tedapat keyword return untuk mengembalikan nilai dari function yang merupakan akhir function. Untuk menjalan kan function panggil nama function dan isi parameter maka function tersebut akan berjalan


Parameter dan Argumen

Parameter dan Argument adalah resource yang dapat di gunakan dalam javascript. Parameter adalah variable terdapat pada () di sebelah nama function berfungsi meyimpan resource yang di kirimkan saat function di panggill. Argument adalah nilai yang di kirimkan ke parameter saat fungsi di panggil. Ketika kondisi parameter lebih sedikit dari argument maka argument berlebih akan di abaikan. Jika kondisi sebaliknya maka nilai dari parameter nya undefined. Arguments adalah variable yang berbentuk array berisi nilai uang di kirimkan saat function di panggil.


Refactoring

Penulisan kode menjadi lebih efisien adalah konsep dari Refactoring, atau dapat juga mengubah kode menjadi lebih baik tanpa mengubah fungsionalitas nya. Refactoring di butuhkan karena agar memudahkan membaca kode program (Readability). Kemudian untuk tidak ada duplikasi kode. Berfungsi juga untuk memudahkan Testing. Dan meningkatan performance dari program. Bermanfaat juga untuk di Kelola dan di kembangkan (Maintain).


Variable Scope

Scope adalah bagaimana sebuah variable dapat di akses dalam program. Terdapat dua buah scope ada Block Scope dan Function Scope. Block Scope adalah sebuah scope dimana variable terdapat di scope tertutup dan tidak dapat di akses oleh scope lain nya terdapat pada bahasa pemrograman C++ dan Java. Function scope adalah variable yang dapat di akes oleh scope lain, bahasa yang menggunakan scope ini adalah Javascriot.


Declaration vs Expression

Function Declaration dan Function Expresion memiliki perbedaan dimana function Declaration wajib mempunyai nama dan Function Expresion tidak wajib mempunyai nama dan juga ketika function ini di simpan di dalam variable. Function declaration dapat di tulis dimanapun misalkan menuliskan sebelum deklarasi atau sebaliknya konsep ini disebut konsep hoisting dan mudah di pahami pemula. Function Expresion harus didefinisakn terlebih dahulu baru di panggil dimana function ini lebih powerfull/kuat.
