Video 1: "FUNCTION pada JAVASCRIPT : Intro"

Function adalah sebuah sub-program/sub-routine yang dapat 'dipanggil' di bagian lain pada program dan merupakan struktur dasar pembentuk JavaScript. Function juga disebut sebagai prosedur (kumpulan statement untuk melakukan tugas atau menghitung suatu nilai). Untuk dapat menggunakannya, kita harus 'membuat' terlebih dahulu function tersebut, lalu 'memanggil'-nya. Function termasuk kedalam First-Class Object.Adapun alasan penggunaan function yaitu:

  1. Reusability (DRY: Don't Repeat Yourself)
  2. Dekomposisi / Abstraksi
  3. Modularitas
Function memiliki 2 kategori yaitu:
  1. Built-in Function: fungsi yang disediakan oleh JavaScript dan kita tinggal 'memanggil'-nya contohnya alert(), prompt(), confirm()
  2. User-defined Function: fungsi yang kita buat sendiri dengan menggunakan keyword function. Pada function terdapat nama function (optional), parameter/argument, dan body. User-defined function dapat mengembalikan nilai (return value) atau tidak. Adapun beberapa cara untuk membuat User-defined Function yaitu:
    • Dengan deklarasi / function declaration yang dapat dibuat dengan sintaks:
      function namaFunction(param1, param2) { body }
    • Dengan ekspresi / function expressions yang dapat dibuat dengan sintaks:
      var namaFunction = function (param1, param2) { body }

Adapun cara untuk memanggil function yaitu dengan menuliskan sintaks:
namaFunction()

Video 2: "FUNCTION pada JAVASCRIPT : Membuat dan Menjalankan FUNCTION"

Secara sederhana, ketika kita bekerja dengan sebuah function biasanya kita akan memiliki nilai yang nantinya akan menjadi bahan/material yang akan diproses oleh function untuk menghasilkan suatu output. Function yang baik adalah function yang mengerjakan satu tugas saja. Blok kode yang dibuat dalam function diperuntukkan suatu untuk tugas spesifik, dapat dipanggil berulang kali (Reusability) serta memudahkan penelusuran.

Video 3: "FUNCTION pada JAVASCRIPT : PARAMETER dan ARGUMENT"

Parameter adalah variabel yang ditulis didalam kurung pada saat function dibuat dan digunakan untuk menampung nilai yang dikirimkan saat function dipanggil. Argument adalah nilai yang dikirimkan ke parameter saat fungsi dipanggil. Untuk lebih jelas dapat dilihat pada contoh kode dibawah.

  1. Contoh penulisan parameter saat membuat function adalah:
    function namaFunction(parameter) { // perintah untuk melakukan tugas terhadap parameter }
  2. Contoh penulisan argument saat memanggil function adalah:
    namaFunction(argument)

Adapun variabel yang bernama arguments yang menyimpan nilai array dari semua argument yang dikirim pada function. Adapun yang terjadi ketika jumlah parameter saat fungsi dibuat berbeda dengan jumlah argument yang dikirim yaitu:

  1. Jika jumlah parameter saat fungsi dibuat lebih banyak dari jumlah argument yang dikirim saat memanggil fungsi maka parameter yang tidak diisi bernilai undefined
  2. Jika jumlah parameter saat fungsi dibuat lebih sedikit dari jumlah argument yang dikirim saat memanggil fungsi maka argument tersebut tetap tersimpan pada fungsi

Video 4: "FUNCTION pada JAVASCRIPT : REFACTORING"

Refactoring adalah sebuah proses mengubah kode agar lebih 'baik' tanpa mengubah fungsionalitasnya. Adapun beberapa faktor mengapa refactoring harus dilakukan yaitu:

  1. Readibility: kemudahan dalam membaca kode program
  2. DRY (Don't Repeat Yourself): duplikasi kode
  3. Testability: penulisan koda agar mudah saat proses pengujian
  4. Performance: bagaimana cara menulis kode yang dapat meningkatkan performa dari program yang dibuat
  5. Maintainability: bagaimada program dapat dengan mudah dikelola atau dikembangkan

Video 5: "FUNCTION pada JAVASCRIPT : VARIABLE SCOPE"

Scope adalah bagaimana cara sebuah variabel dapat diakses dalam program. Dalam bahasa pemrograman terdapat dua jenis scope yaitu:

  1. block scope: variabel yang dibuat didalam block scope tidak dapat digunakan diluar dari block-nya. block scope digunakan pada bahasa pemrograman seperti C dan Java
  2. function scope: variabel yang dibuat didalam block scope dapat digunakan diluar dari block-nya. block scope digunakan pada bahasa pemrograman seperti Javascript

Dalam Javascript, ketika variabel dibuat tanpa dibungkus scope disebut global scope atau window scope. Global scope atau window scope dapat diakses pada block scope manapun. Deklarasi variabel didalam block scope atau saat menjadi parameter dengan nama yang sama dengan variabel global scope akan menjadi 2 variabel yang berbeda.

Video 6: FUNCTION pada JAVASCRIPT : DECLARATION vs. EXPRESSION

Rumus dari Function Declaration yaitu:
function namaFunction(listParameter opt) { FunctionBody }

Rumus dari Function Expression yaitu:
function namaFunctionopt (listParameteropt) { FunctionBody }

Adapun perbedaan dari Function Declaration dan Function Expression yaitu:

  • Function Declaration:
    1. Lebih fleksibel (dapat ditulis dimanapun) karena pada javascript ada konsep Hoisting dimana deklarasi sebuah function pasti disimpan terlebih dahulu didalam memory
    2. Mudah dipahami pemula
  • Function Expression:
    1. Harus didefinisikan terlebih dahulu sebelum dipanggil
    2. Lebih powerfull:
      • Sebagai closure
      • Sebagai argumen untuk function lain
      • IIFE (Immediately Invoked Function Expression)
