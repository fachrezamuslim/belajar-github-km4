Apa itu Function?

Javascript dibentuk dari kumpulan function, yang mana Function sendiri merupakan sebuah sub-program yang dapat dipanggil di bagian lain pada program. Function juga disebut sebagai procedure atau kumpulan statement yang dapat melakukan tugas tertentu atau menghitung sebuah nilai.

Function dibuat untuk mengatasi beberapa permasalahan yaitu:

  • Meningkatkan Reusability dalam program yang dibuat (DRY)
  • Mengurangi Kompleksitas dari program yang dibuat
  • Implementasi modularitas dalam program sehingga memudahkan dalam debugging

Function sendiri dapat dikategorikan menjadi 2, yaitu:

  • Built-in Function

    Function yang sudah disediakan/dibuat oleh javascript yang sudah ready untuk dipanggil/digunakan. Contohnya: alert, confirm, dan prompt.

  • User-Defined Function

    Function yang dibuat oleh programmer itu sendiri. User defined function dibuat dengan menggunakan keyword "function".

    User-defined function dapat memiliki nama function (optional) dan dapat memiliki parameter yang disimpan dalam kurung, dan dipisahkan dengan koma apabila lebih dari 1. Function body disimpan dalam kurung kurawal dan didalamnya dapat memiliki return value atau tidak.

    //Function declaration

    function Add(num1, num2) {

    let result = num1 + num2;

    return result;

    }


    //Function dipanggil

    console.log(Add(3+4)); //output: 4

    Function dapat disimpan kedalam variable sehingga dapat dipanggil dengan nama variable tersebut. Contohnya adalah sebagai berikut:

    //Function declaration

    const Add = function(num1, num2) {

    let result = num1 + num2

    return result

    }


    //Function dipanggil

    console.log(Add(3+4)); //output: 4

Membuat Function

Function biasanya bekerja dengan menerima sebuah input terlebih dahulu yang kemudian diolah kedalam function yang nantinya akan menghasilkan suatu output tertentu.

"function yang baik hanya mengerjakan 1 hal spesifik saja"

Parameter dan Argument

Parameter merupakan variabel yang ditulis dalam kurung pada saat function dibuat. Parameter ini digunakan untuk menampung nilai yang dikirimkan yang akan dipanggil pada saat function dipanggil.

Argument merupakan nilai yang dikirimkan ke parameter pada saat function dipanggil. Argument dapat dikirim langsung sebagai value, atau bisa juga nilai argument disimpan kedalam variable terlebih dahulu baur kemudian dikirim sebagai argument. Argument dapat diberi operasi apapun, misalnya expresi aritmatika, parsing data tipe, dsb.

Parameter and Argument Behaviors

  • jika parameter lebih sedikit dari argument maka argument kelebihannya akan diabaikan.
  • jika argument lebih sedikit dari parameter maka parameter kelebihannya akan diberi nilai default. Nilai default dapat ditentukan spesific atau secara default nilainya adalah undifined.

Function sebenarnya memiliki sebuah pseudo element yang disebut dengan arguments. Arguments akan menampung semua argument yang dikirimkan saat function dipanggil.

Refactoring

Refactoring merupakan sebuah proses untuk mengubah kode yang kita tulis agar menjadi lebih baik lagi tanpa mengubah fungsionalitas dari program itu sendiri.

Alasan dilakuakn refactoring

  • Meningkatkan code readability atau kemudahan dalam membaca kode program. Program yang baik adalah program yang mudah dibaca, tidka hanya oleh yang membuat tetapi juga dapat dipahami oleh orang lain.
  • Implementasi DRY (Do not Repeat Yourself), yaitu penggunaan bagian kode secara efisien misalnya penggunaan variable, function, method, dsb.
  • Testability, yaitu penulisan kode yang baik agar mudah saat dilakukan testing
  • Meningkatkan performance pada program, misalnya pemilihan tipedata yang tepat, alokasi memory yang efektif, dsb.
  • Meningkatkan maintainabillity, yaitu bagaimana program dapat dengan mudah dikelola dan dikembangkan

Variable Scope

Scope marupakan sebuah istilah yang artinya lingkup variable, atau bagaimana sebuah variable dapat diaksess dalam program.

Konsep Scope

  • Block Scope

    Block scope digunakan oleh bahasa pemrograman seperti C dan Java. Variable yang disebut block scope merupakan variable yang dideklarasikan didalam kurung kurawal "{ ... }".

  • Function Scope

    variable yang mengimplementasikan function scope merupakan variable yang dideklarasikan didalam function dan hanya dapat diakses dalam function tersebut saja.

Behavior variable pada javascript

  • Variable yang dideklarasikan pada object terluar (window) pada program disebut sebagai global scope atau window scope yang dapat digunakan dimanapun pada program.
  • Variable yang dideklarasikan didalam sebuah function disebut dengan function scope atau local scope. Variable ini hanya dapat diakses pada function tersebut saja.
  • Variable pada function scope dapat memiliki nama yang sama dengan global variable. Jika ini terjadi maka, apabila nama variable dipanggil maka nilai yang dikeluarkan akan sesuai dengan nilai yang dideklarasikan pada scope masing-masing. (local variable tidak menimpa global variable).
  • Apabila didalam function terdapat value assignment kedalam variable tanpa ada decaration keyword (var, let, const) maka nilai global variable akan ditimpa.
  • Apabila pada window tidak ada declaration keyword, maka javascript akan otomatis membuat declaration keyword pada global scope. Untuk mencegah hal ini terjadi kita dapat menggunakan keyword "use strict".
  • Parameter dan argument memiliki sifat local scope pada function.

Declaration vs Expression

Function declaration :

function identifier (ParameterList opt) {FuntionBody}

Function expression :

function identifier opt (ParameterList opt) {FuntionBody}

Function expression baru dapat disebut sebagai function expression apabila sebuah function disimpan kedalam sebuah variable. Function expression memeiliki beberapa sifat :

  • Lebih Flexible (dapat ditulis dimanapun) karena javascript memiliki konsep hoisting (deklarasi sebuah function akan disimpan terlebih dahulu dalam memory).
  • Cenderung lebih mudah dipahami oleh pemula

Function expression harus didefinisikan terlebih dahulu sebelum dipanggil. Funtion expression lebih powerfull dibanding funtion declaration, karena dapat digunakan sebagai closure, dapat digunakan sebagai argument untuk funtion lain, dan dapat digunakan untuk IIFE (Immediately Infoked Funtion Expression).
